## 四、RALPLAN-DR 决策摘要

### 核心原则 (Principles)

1. **算法优先验证**：HLS 实现前必须完成算法仿真与定点化确认
2. **Pipeline 效率最大化**：设计以 1 pixel/clock 吞吐为目标
3. **资源-性能平衡**：在满足时序前提下优化资源使用
4. **可测试性设计**：预留调试接口与测试点
5. **文档同步交付**：代码与文档同步更新

### 决策驱动因素 (Decision Drivers)

| 驱动因素 | 影响程度 |
|----------|----------|
| **算法复杂度** | 决定 HLS 优化策略与资源预算 |
| **吞吐量需求** | 决定 Pipeline 设计深度 |
| **目标器件** | 决定可用资源与时序约束 |

### 可行选项分析 (Viable Options)

#### 选项 A：纯 HLS 实现
**描述**：全部模块使用 HLS C++ 实现

| 优点 | 缺点 |
|------|------|
| 开发周期短 | 部分 RTL 优化受限 |
| 易于算法迭代 | 资源效率可能不如手工 RTL |
| 可维护性好 | 复杂控制逻辑可能难综合 |

**适用场景**：算法明确、时间紧迫、资源预算充足

---

#### 选项 B：HLS + RTL 混合实现
**描述**：核心数据通路用 HLS，控制逻辑用 SystemVerilog

| 优点 | 缺点 |
|------|------|
| 资源效率更高 | 开发周期更长 |
| 控制逻辑更灵活 | 需要更多验证工作 |
| 更好的时序控制 | 接口设计更复杂 |

**适用场景**：资源受限、复杂控制、高性能要求

### 6.3 各阶段详细设计

#### Stage 1: Sobel 5x5 滤波

**Sobel_X Kernel (5x5):**
```
[-1  -2   0   2   1]
[-4  -8   0   8   4]
[-6 -12   0  12   6]
[-4  -8   0   8   4]
[-1  -2   0   2   1]
```

**Sobel_Y Kernel (5x5):**
```
[-1  -4  -6  -4  -1]
[-2  -8 -12  -8  -2]
[ 0   0   0   0   0]
[ 2   8  12   8   2]
[ 1   4   6   4   1]
```

**输出:** `Gradient = |Gx| + |Gy|` (L1 范数，适合硬件实现)

#### Stage 2: 自适应窗口选择

| 梯度范围 | 窗口大小 | 应用场景 |
|----------|----------|----------|
| [0, Th0) | 2x2 | 平坦区域 |
| [Th0, Th1) | 3x3 | 轻微纹理 |
| [Th1, Th2) | 4x4 | 中等纹理 |
| [Th2, ∞) | 5x5 | 强边缘区域 |

**阈值寄存器:** `sobel_thresh_0/1/2/3` (可配置)

#### Stage 3: 梯度加权平均滤波

**核心算法:**
```
Grad_u, Grad_d, Grad_l, Grad_r, Grad_c = invSort(Grad_u, Grad_d, Grad_l, Grad_r, Grad_c)
Grad_sum = Grad_u + Grad_d + Grad_l + Grad_r + Grad_c

         ⎧ (avg_u * Grad_u + avg_d * Grad_d + avg_l * Grad_l + avg_r * Grad_r + avg_c * Grad_c) / Grad_sum   (Grad_sum ≠ 0)
avg =    ⎨
         ⎩ (avg_u + avg_d + avg_l + avg_r + avg_c) / 5                                                       (Grad_sum = 0)
```

**权重来源:**
- 权重 **不是** 通过寄存器配置
- 权重来自 **Stage 1 计算的梯度值** (Grad_u, Grad_d, Grad_l, Grad_r, Grad_c)
- 分别对应四个方向邻近像素（上、下、左、右）及中心像素位置的梯度值

**方向定义:**
- **avg_c**: 中心像素平均值 (2x2_avg_center)
- **avg_u**: 上方区域平均值 (2x2_avg_up)
- **avg_d**: 下方区域平均值 (2x2_avg_down)
- **avg_l**: 左侧区域平均值 (2x2_avg_left)
- **avg_r**: 右侧区域平均值 (2x2_avg_right)

**梯度对应关系:**
- **Grad_c**: 中心位置梯度值
- **Grad_u**: 上方邻近像素位置梯度值
- **Grad_d**: 下方邻近像素位置梯度值
- **Grad_l**: 左侧邻近像素位置梯度值
- **Grad_r**: 右侧邻近像素位置梯度值

**invSort 说明:**
- 对五个梯度值进行逆序排序
- 用于权重归一化处理

**关键设计要点:**
- ⚠️ **梯度数据需要存储在 Line Buffer 中**，供 Stage 3 使用
- 梯度值由 Stage 1 计算产生
- **行延迟机制**: Stage 1 先计算梯度并存储 5 行，Stage 2/3 延迟若干行后处理
- 延迟处理后，"下方"(Grad_d) 和 "右侧"(Grad_r) 的梯度数据已在 Line Buffer 中可访问
- Line Buffer 需额外存储 **5 行梯度数据**

#### Stage 4: 行间 Blending

**公式:** `Output = α × AvgOutput + (1-α) × PrevLine[x]`

**系数寄存器:** `blend_coeff` (0-255, 128 = 50% blend)

#### Stage 5: 融合写回

**功能:** 将滤波输出与原始数据融合后写回 src Line Buffer

**融合公式:**
```
out = Σ(src_i × coeff_i) / Σ(coeff_i)
```

**窗口大小与融合范围:**

| winSize | 融合范围 | 系数配置 |
|---------|----------|----------|
| 2x2, 3x3 | 3x3 (5x5 抽头，外围一圈系数=0) | 内圈 9 个系数通过寄存器配置 |
| 4x4, 5x5 | 5x5 | 25 个系数全部可配置 |

**系数寄存器:**
- `blend_coeffs_2x2[9]`: 2x2 窗口融合系数（3x3 内圈）
- `blend_coeffs_3x3[9]`: 3x3 窗口融合系数（3x3 内圈）
- `blend_coeffs_4x4[25]`: 4x4 窗口融合系数（5x5 全窗口）
- `blend_coeffs_5x5[25]`: 5x5 窗口融合系数（5x5 全窗口）

**5x5 窗口系数布局:**
```

├───┼───┼───┼───┼───┤      ├───┼───┼───┼───┼───┤
│ 0 │c11│c12│c13│ 0 │      │c10│c11│c12│c13│c14│
├───┼───┼───┼───┼───┤      ├───┼───┼───┼───┼───┤
```

**UV Line Buffer 设计 (6行复用):**
- **单 Buffer 多功能设计**: 6 行 Line Buffer 同时支持多种操作
- **双端口 SRAM**: 支持同时读取 (Sobel) 和写入 (新像素/融合结果)
- 反压机制处理写入冲突

```

│                 UV Line Buffer (6行) 复用架构                │
├─────────────────────────────────────────────────────────────┤
│                                                             │

│   Row 1 ───┤                                                │
│   Row 2 ───┼─ Sobel 5x5 窗口 (中心)                         │
│   Row 3 ───┤                                                │
│   Row 4 ───┴─ Sobel 5x5 窗口 (下边界)                       │
│             │                                               │
│             ├─ Stage 5 融合写回目标                         │
│             └─ Stage 4 Blending 的 PrevLine 来源            │
│                                                             │
│   Row 5 ─── 新像素写入行 (实时接收输入)                      │
│                                                             │
│   Port A (读): Sobel 5x5 滑动窗口                           │
│   Port B (写): 新像素写入 / 融合结果写回                     │
│                                                             │
│   反压: Port B 写入冲突时暂停前级输入                        │
│                                                             │

```

**Line Buffer 滚动更新时序:**
```
时间 →
        ┌─────────────────────────────────────────────────────────────┐
Row N   │ 存储: Row 0                                                  │
        │ 功能: Sobel 窗口第1行 + 融合写回目标 + 下一行 Blending 源    │
        └─────────────────────────────────────────────────────────────┘
        ┌─────────────────────────────────────────────────────────────┐
Row N+1 │ 存储: Row 1 (原 Row 0 内容滚出，或被融合结果覆盖)           │
        │ 功能: Sobel 窗口第2行                                        │
        └─────────────────────────────────────────────────────────────┘
        ...
        ┌─────────────────────────────────────────────────────────────┐
Row N+5 │ 存储: Row 5                                                  │
        │ 功能: 接收新输入像素                                          │
        └─────────────────────────────────────────────────────────────┘
        ┌─────────────────────────────────────────────────────────────┐
下一帧  │ Row 0-4 已被融合结果更新，可供下一行 Blending 使用           │
        └─────────────────────────────────────────────────────────────┘
```

### 6.4 Line Buffer 需求

| 阶段 | Line Buffer 深度 | 数据类型 | 说明 |
|------|------------------|----------|------|
| Sobel 5x5 + 融合写回 | 6 行 | UV 像素数据 | 5 行 Sobel 窗口 + 1 行新数据写入；融合写回直接更新此 Buffer |
| 梯度存储 | 5 行 | 梯度值 (Grad) | Stage 1 输出，供 Stage 3 加权使用 |

**总计: 11 行 Line Buffer**
- UV 像素数据 (Sobel + 融合写回): 6 行
- 梯度数据: 5 行

**优化设计要点:**

1. **UV Line Buffer 多功能复用 (6行):**
   - Row 0-4: Sobel 5x5 滑动窗口
   - Row 5: 新像素实时写入行
   - Stage 5 融合写回结果直接更新 Row 0-4
   - Stage 4 Blending 的 PrevLine 直接从 Row 0-4 读取（无需额外 Blending Line Buffer）

2. **反压机制:**
   - 当 Row 5 写入新像素与融合写回操作冲突时，反压前级暂停输入
   - 冲突仅发生在同一列位置同时需要写入新数据和融合结果
   - 反压周期数极少，不影响整体吞吐量

3. **省略 Blending Line Buffer 的原因:**
   - Stage 5 融合写回后的数据存储在 UV Line Buffer 中
   - 下一行处理时，UV Line Buffer 中的数据即为"上一行输出"
   - Stage 4 Blending 可直接从 UV Line Buffer 获取 PrevLine[x]

**资源对比:**
| 方案 | UV 行数 | 梯度行数 | Blending | 总计 |
|------|---------|----------|----------|------|
| 原方案 (双缓冲) | 15 行 | 5 行 | 1 行 | 21 行 |
| **优化方案** | **6 行** | 5 行 | 0 行 | **11 行** |

**节省: 10 行 (48%)**

### 6.4.1 行延迟机制说明

**流水线时序:**
```
时间 →
        ┌─────────────────────────────────────────────────────────────┐
Row N   │ Stage1: 计算梯度 → 存入 Grad Line Buffer                    │
        └─────────────────────────────────────────────────────────────┘
        ┌─────────────────────────────────────────────────────────────┐
Row N+1 │ Stage1: 计算梯度 → 存入 Grad Line Buffer                    │
        └─────────────────────────────────────────────────────────────┘
        ┌─────────────────────────────────────────────────────────────┐
Row N+2 │ Stage1: 计算梯度 → 存入 Grad Line Buffer                    │
        └─────────────────────────────────────────────────────────────┘
        ...
        ┌─────────────────────────────────────────────────────────────┐
Row N+4 │ Stage1: 计算梯度 → 存入 Grad Line Buffer (此时 Buffer 满)   │
        └─────────────────────────────────────────────────────────────┘
        ┌─────────────────────────────────────────────────────────────┐
Row N+5 │ Stage1: 计算梯度                                             │
        │ Stage2/3: 对 Row N 进行滤波 (可访问 N-2~N+2 共5行梯度)       │
        └─────────────────────────────────────────────────────────────┘
```

**关键点:**
- Stage 1 领先 Stage 2/3 若干行（约 4-5 行延迟）
- 当 Stage 3 处理当前像素时，上、下、左、右、中心位置的梯度值都已在 Grad Line Buffer 中
- Grad_d (下方) 和 Grad_r (右侧) 在延迟处理后变为可访问的"历史"数据
